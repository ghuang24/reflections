When would you want to use a remote repository rather than keeping all your work
local?

    When you want others to contribute to the project as well
    When you want to work on files from multiple locations

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    You fetch versions of the programs at logical intervals so you can always have a backup.

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

    fork: getting a repository from online while giving original credit
    clone: getting a repository online w/o the credit
    branch: creating test versions of current project

    fork/clone for getting code from online
    clone more for self

What is the benefit of having a copy of the last known state of the remote
stored locally?

    You can use it as backup
